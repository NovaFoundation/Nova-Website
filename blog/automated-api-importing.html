WORKING WITH EXTERNAL APIS

When dealing with multiple targets you are often required to call upon external APIs. There are a couple ways you could do this:

  • Call them directly with external code blocks
  • Create bindings to the API in Nova

CALLING EXTERNAL CODE DIRECTLY

The quickest (and dirtiest) way of working with external libraries is to call the external code directly. Nova provides external blocks that are used to achieve this functionality:

myFunc() { external c { printf("IN EXTERNAL C!\n"); } }

This is really only helpful if you can pass variables and functions to and from the external code. This example shows how to pass a variable to the external code:

myFunc() { Int myInt = 100 external c { // The number is 100! printf("The number is %d!\n", #{myInt}); } }

Passing variables from external code to Nova is done by assigning the Nova variable inside the external code:

myFunc() { Int myInt external c { #{myInt} = 100; // The number is 100! printf("The number is %d!\n", #{myInt}); } // The number is 100 in Nova too! Console.writeLine("The number is #myInt in Nova too!") }

You can also run Nova statements inside external code:

getMyInt() -> Int { Int myInt external c { #{myInt} = 100 #{return myInt * 2}; } }

CREATING BINDINGS TO API

Calling external code through external blocks is the basis of how creating bindings to an API works in Nova. When creating an API for a language, you typically do it by hand. For example, if you were to create bindings for some std output functions in C, you could do it like:

public atoi(String s) -> Int { external c { return atoi(#{s.chars.data}); } } public random() -> Long { external c { return rand(); } } public currentTimeMillis() -> Long { external c { return current_time_millis(); } }

This creates bindings to the atoi, random, and current_time_millis functions in C. Doing this by hand can be very tedious and susceptible to errors. Because of this, automated importing of APIs would be immensely useful.

AUTOMATED API IMPORTING

Because you can write custom parsers, adding API importing functionality will be surprisingly simple. If a parser for a language is available, instead of translating it to Nova, you could pass an -api argument to the compiler to have it generate the API binding for the functions in the input library. For example, say you want to import the C standard library. Using a C header file parser, you could import all stdlib functions with this command:

novac path/to/c/stdlib -input c-header -api -d nova-c-stdlib

This would output the translated API binding for the c stdlib into a folder named nova-c-stdlib. You could then reference this binding in your project like the following:

import static "stdlib" class MyClass { public static main(String[] args) { let path = getenv("PATH") let myStringPath = new String(path) Console.writeLine("Path: #myStringPath") } }

The first line import static "stdlib" imports our API generated binding's static functions (all C functions are considered static in the context of Nova). Then we use the external binding at let path = getenv("PATH"). Because getenv returns a native char array, you have to convert it to a String before you can output it as one at Console.writeLine("Path: #myStringPath").

To include the API binding in your project's compilation process, you would simply include nova-c-stdlib as one of your compilation sources:

novac MyProject nova-c-stdlib -o MyExecutable

This will compile your project and output your executable as MyExecutable.

CONCLUSION

Automated importing of external APIs plays an integral part in making compilation to multiple-targets more powerful. It alleviates many of the issues that normally come up when writing programs that interface with multiple different languages. Automated importing of external APIs expidiates the process of multi-target development, while making it safe. Both of which are fundamental to the design of the Nova language.
