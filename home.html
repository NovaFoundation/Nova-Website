NOVA SYNTAX

Trees and Lists

Nova's List data structure makes processing lists of data easy. With the use of lambda expressions, the code to perform operations can be concise. Like many other modern languages, Nova lets you use lambda expressions and functions as first class values. Nova is smart enough to infer the type of the variables that were defined with the use of the "let" and "var" modifiers.

In the code example on the right, you can see string interpolation in action. The strings passed to Console.writeLine() show this. String interpolation makes injecting variable values into strings effortless. You just use #{ expression } (or even just #variable, if it is just accessing an attribute) within a string literal and the compiler does the rest for you.

One cool thing about the way List's are implemented in Nova is that because it is a trait, any class can implement it and they only have to define an Iterator for the class for it to gain access to all of the functional capabilities of a List.

let tree = new BinaryTree(["F", "B", "A", "D", "C", "E", "G", "I", "H"])


Console.writeLine("Preorder: #{tree.preorder().join(", ")}")

// Preorder: [F, B, A, D, C, E, G, I, H]

Console.writeLine("Inorder: #{tree.inorder().join(", ")}")

// Inorder: [A, B, C, D, E, F, G, H, I]

Console.writeLine("Postorder: #{tree.postorder().join(", ")}")

// Postorder: [A, C, E, D, B, H, I, G, F]

Console.writeLine("Levelorder: #{tree.levelorder().join(", ")}")

// Levelorder: [F, B, G, A, D, I, C, E, H]

Class types

Nova contains three different types of class structures:

  • Classes
  • Interfaces
  • Traits

Each of these has their own strengths and weaknesses. For instance, you can only inherit from one class at most. In contrast, you can inherit from as many traits and interfaces as you want. Because of this, traits and interfaces cannot have formal constructors. Traits differ from interfaces in such that traits can define method bodies and class properties. Interfaces are just like traditional Java interfaces; they only contain abstract method declarations.

Using the code shown, you can create an instance of MyClass with an instantiation as such: let myObj = new MyClass(). Then you can call any of the functions defined in the class or its ancestors (MyTrait and MyInterface). You could call:

  • myObj.saySomething() which would output My value is: 'my value'
  • myObj.calculateValue() which returns a String "my value"
  • myObj.plus(5) which adds 5 to myObj.id and returns myObj

class MyClass implements MyTrait, MyInterface {

public calculateValue() => "my value"


public saySomething() {

Console.writeLine("My value is: '#calculateValue()'")

}

}


trait MyTrait implements PlusOperator<Int, MyTrait> {

visible Int id


public plus(Int num) -> MyTrait {

id += num

return this

}

}


interface MyInterface {

public calculateValue() -> String

}

NOVA COMPILATION TARGETS

NOVA -> JAVASCRIPT

The Nova compiler generates Javascript code with a similar syntactical structure to the original source code. When using Nova to write javascript code you get the benefit of a statically typed language with compile-time optimizations without losing the versatility of Javascript. This makes it safe and natural to write libraries in Nova and use them as libraries in plain Javascript.

public static main() {

let names = ["Braden", "Ethan", "George"]


for (name in names) {

Console.writeLine("Hello, #name!")

}

}

var names = new NovaArray(["Braden", "Ethan", "George"]);

var iterator = names.iterator();


while (iterator.hasNext()) {

var name = iterator.next();

console.log("Hello, " + name + "!");

}

NOVA -> JAVA

Nova's syntax takes a lot from Java's syntax. Because of this, the generated Java code is readable and works with existing Java libraries seamlessly. This allows for writing libraries for anything that deals with Java code (e.g., servers and mobile development). For instance, you can write Android compatible code in Nova. A benefit of writing Android compatible code in Nova is that when the Swift compilation target is written, you will be able to compile existing compatible code to both Android and Swift. Essentially you get the best of both worlds.

public static main() {

let names = ["Braden", "Ethan", "George"]


for (name in names) {

Console.writeLine("Hello, #name!")

}

}

public static void main(String[] args) {

Array<String> names = new Array<>(new String[] { "Braden", "Ethan", "George" });

Iterator<String> iterator = names.iterator();


while (iterator.hasNext()) {

String name = iterator.next();

System.out.println("Hello, " + name + "!");

}

}

NOVA -> C

Despite being a notably different paradigm, the Nova C compiler is currently the most mature of the compiler targets. The ability to compile to C allows compilation on a vast amount of high-level and low-level devices. All that is required is a C99 compiler to compile the generated C code.

public static main() {

let names = ["Braden", "Ethan", "George"]


for (name in names) {

Console.writeLine("Hello, #name!")

}

}

int main(int argc, char** argvs) {

nova_Array* names = generate_array();

nova_ArrayIterator* iterator = nova_Array_iterator(names, exceptionData);


while (nova_ArrayIterator_hasNext(iterator, exceptionData)) {

nova_String* name = (nova_String*)nova_ArrayIterator_next(iterator, exceptionData);

nova_Console_writeLine(nova_String_concat(nova_String_concat(nova_String_construct(0, exceptionData, "Hello, "), exceptionData, name), nova_String_construct(0, exceptionData, "!")));

}


return 0;

}


nova_Array* generate_array() {

nova_String** temp = (nova_String**)NOVA_MALLOC(sizeof(nova_String*) * 3);


temp[0] = nova_String_construct(0, exceptionData, "Braden");

temp[1] = nova_String_construct(0, exceptionData, "Ethan");

temp[2] = nova_String_construct(0, exceptionData, "George");


return nova_Array_construct(0, exceptionData, (nova_Object**)temp, 3);

}
